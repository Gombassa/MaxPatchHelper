Bugs & Crashes

#1 Bug Blocking requests.post inside async def functions ✓ Fixed server.py : 457–476, 530–544

run_websocket_learning_stage and the spec-extraction block in run_websocket_generation_stage call requests.post(…) synchronously inside async def functions. This blocks the entire asyncio event loop — no other requests can be handled while Ollama is thinking. 

# BAD — freezes the event loop
async def run_websocket_learning_stage(websocket, session_history):
    response = requests.post(OLLAMA_CHAT_URL, ...)   # synchronous, blocks loop

# FIX — offload to a thread
async def run_websocket_learning_stage(websocket, session_history):
    loop = asyncio.get_event_loop()
    response = await loop.run_in_executor(
        None, lambda: requests.post(OLLAMA_CHAT_URL, ...)
    )
#2 Bug Fire-and-forget task in extract_and_send_spec ✓ Fixed server.py : 406–432

This sync function calls asyncio.create_task(websocket.send_json(…)) and returns immediately. The task may never complete if the connection drops right after. The function should be async def and awaited by the caller. 

# BAD — unwaited task, invisible on disconnect
def extract_and_send_spec(websocket, text):
    asyncio.create_task(websocket.send_json({...}))   # fire and forget

# FIX
async def extract_and_send_spec(websocket, text):
    await websocket.send_json({...})
#3 Bug Background thread leaks on WebSocket disconnect ✓ Fixed server.py : 355–403

run_websocket_llm_turn spawns a background thread with no stop event. On WebSocketDisconnect, the outer handler exits but the thread keeps streaming into the queue indefinitely, wasting CPU and Ollama resources. Pass a threading.Event into the thread and check it in the streaming loop.

#4 Bug Format injection via user input (str.format on untrusted strings) ✓ Fixed explain.py : 177 · generate.py : 174 · guided.py : 199

All three prompt builders call TEMPLATE.format(…, query=query_text) where query_text is raw user input. If the user types a query containing { or } (e.g. "how does {dict} work?"), Python raises KeyError at runtime and crashes the request. 

# BAD
prompt = TEMPLATE.format(query=user_input)   # crashes if input has { or }

# FIX — build the template without the query slot, then concatenate
prompt = TEMPLATE.format(context=ctx, ...) + f"\nUSER QUERY:\n{user_input}"
#5 Bug MSP has_dsp check misses newobj-wrapped audio objects ● Open validate.py : 426–428

has_dsp = any(b.maxclass.endswith("~") for b in boxes) only checks maxclass. A cycle~ written as {"maxclass": "newobj", "text": "cycle~"} has maxclass == "newobj", so .endswith("~") is false. The existing has_object() helper already handles this correctly — use it. 

# BAD
has_dsp = any(b.maxclass.endswith("~") for b in boxes)

# FIX — use has_object() which checks both maxclass and newobj text
has_dsp = has_object(["cycle~", "gain~", "sig~", "filter~", "dac~", "adc~"])

Performance

#6 Perf New ChromaDB client created on every query ● Open retrieve.py : 33

chromadb.PersistentClient(path=CHROMA_DB_PATH) is instantiated inside query_vector_db, so every RAG lookup opens the database from disk, initialises WAL, etc. Cache it as a module-level singleton. 

# BAD — paid on every call
def query_vector_db(...):
    chroma_client = chromadb.PersistentClient(path=CHROMA_DB_PATH)

# FIX — module-level singleton
_chroma_client = None
def _get_client():
    global _chroma_client
    if _chroma_client is None:
        _chroma_client = chromadb.PersistentClient(path=CHROMA_DB_PATH)
    return _chroma_client
#7 Perf Inlet/outlet index JSON re-read from disk per object ● Open validate.py : 90–99

get_true_ports() opens and fully parses inlet_outlet_index.json for every object not in TRUSTED_PORT_COUNTS. A patch with 10 unknown objects triggers 10 file reads of the same file. Cache the parsed dict at module load.

Security

#8 Security CORS wildcard combined with allow_credentials=True ● Open server.py : 41–47

Browsers reject Access-Control-Allow-Origin: * when credentials are included. This misconfiguration will silently break cookie/auth scenarios and is flagged as a security misconfiguration by OWASP. Restrict the origin to your actual frontend URL or drop allow_credentials. 

# BAD
allow_origins=["*"],
allow_credentials=True,     # invalid combination

# FIX — pin to your dev frontend
allow_origins=["http://localhost:5173"],
allow_credentials=True,

Maintainability

#9 Maint validate.py duplicates BASE_DIR / INDEX_PATH from config.py ● Open validate.py : 6–7

These constants are already defined in config.py. Importing them keeps a single source of truth and prevents the two copies drifting apart.

#10 Maint Hardcoded relative example-patch paths in generate.py ● Open generate.py : 11–12

EXAMPLE_MAX_PATCH_PATH = "data/example_patches/…" is relative to the current working directory. It breaks if the script is run from anywhere other than the repo root. Use DATA_DIR from config.py. 

# BAD
EXAMPLE_MAX_PATCH_PATH = "data/example_patches/max/sine_generator.json"

# FIX
from config import DATA_DIR
EXAMPLE_MAX_PATCH_PATH = os.path.join(DATA_DIR, "example_patches", "max", "sine_generator.json")
#11 Maint sys.path.insert hacks in every entry point ● Open assistant.py · server.py · tests/

Every file that imports from assistant/ manually injects the path at runtime. The clean fix is to add assistant/__init__.py and install the package in editable mode (pip install -e .), letting Python's import system do its job.

#12 Maint Learning-stage and spec-extraction logic duplicated across CLI and server ● Open guided.py : 261–375 · server.py : 434–564

run_learning_stage (CLI) and run_websocket_learning_stage (server) are near-identical. Same for spec extraction. Both sets share a prompt and Ollama call but differ only in how output is delivered. Extract the shared Ollama logic into a core function and call it from both wrappers.

Minor / Quality

#13 Minor Stale model name in debug print ● Open guided.py : 378

Print says qwen2.5-coder:14b but config.py sets GENERATE_MODEL = "qwen2.5-coder:7b". Replace the string literal with {GENERATE_MODEL}.

#14 Minor "device" keyword causes M4L false positives ● Open explain.py : 91

"device" is in m4l_keywords, so a question like "what does the device message do?" triggers full LOM schema loading in a plain Max context. Remove it or require a more specific phrase like "live device".

#15 Minor Context window overflow risk in learning stage ● Open guided.py : 279–283 · server.py : 453–455

The entire session history is concatenated raw into the summary prompt with no length check. A long session easily overflows GUIDED_CONTEXT_WINDOW = 4096 tokens. Truncate or summarise earlier turns before appending.

#16 Minor Busy-poll pattern in SSE and WebSocket token streams ● Open server.py : 155–163, 391–402

q.get_nowait() / asyncio.sleep(0.01) polls 100×/second. Switch to asyncio.Queue with await q.get() to yield only when a token actually arrives.
