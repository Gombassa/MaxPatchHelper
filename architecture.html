v1.0 post-review June 2026 9 modules · 1 server · 9 data files
1

Core Principle

Every file in assistant/ owns exactly one concern. No file duplicates a constant, function, or responsibility owned by another. When you need to add something, the first question is always which file owns this? — not which file is convenient to put it in.

The three rules that follow from this:

Rule 1: Constants defined in config.py are imported everywhere else. Never redefined. Rule 2: Prompt strings live in prompts.py only. Imported by explain, generate, guided. Never inline. Rule 3: No file imports from server.py. Server imports from everything; nothing imports from server.

Module Map

Nine modules plus the server. Each card shows what the file owns, what it is allowed to import, and its category.

config.py Config
Owns
All constants: paths, URLs, model names, context window sizes. Single source of truth. No logic, no functions.
os nothing from assistant/
prompts.py Prompts
Owns
All system prompt strings: EXPLAIN_SYSTEM_PROMPT, GENERATE_SYSTEM_PROMPT (includes full inlet/outlet reference table), GUIDED_SYSTEM_PROMPT. No logic, no functions.
nothing — no imports
retrieve.py Retrieval
Owns
ChromaDB queries. Embedding generation via Ollama. Returns raw document chunks to callers. Manages the ChromaDB client (should be singleton — see §7).
config chromadb requests
classify.py Classifier
Owns
Intent classification only: returns EXPLAIN, GENERATE, or GUIDED for a given query. Single Ollama call, zero temperature. No retrieval, no prompts.
config requests json
explain.py LLM · Explain
Owns
Explain mode inference. Also owns shared utility functions used by generate and guided: load_inlet_outlet_index(), load_lom_schema(), detect_m4l_context().
config prompts retrieve requests
generate.py LLM · Generate
Owns
Patch generation inference. 3-attempt validation loop. M4L device type detection. Example patch loading. Returns structured result dict with valid/patch/errors.
config prompts retrieve explain validate
validate.py Validation
Owns
All patch validation logic. Pydantic models for .maxpat schema. TRUSTED_PORT_COUNTS table. M4L device checks. Returns {valid, errors, warnings}. Never calls Ollama.
config pydantic json nothing else from assistant/
guided.py LLM · Guided
Owns
Guided build mode. Personal idioms file management (load, append, gitignore check). ENRICHED_USER_TEMPLATE. CLI REPL. WebSocket core logic (shared with server via extraction — see §7).
config prompts retrieve explain generate
server.py Entry · API
Owns
FastAPI app. HTTP and WebSocket endpoints. SSE streaming. Async wrappers (_ollama_post helper). CORS config. Routes requests to explain/generate/guided. Imports from all assistant modules. Nothing imports from server.
config explain generate validate guided retrieve fastapi
Note on explain.py utility functions: load_inlet_outlet_index(), load_lom_schema(), and detect_m4l_context() are shared utilities that currently live in explain.py and are imported by generate.py and guided.py. This is a pragmatic arrangement that works. A future refactor could move them to a dedicated context.py utility module, but do not do this without explicit planning — it requires updating imports in three files.

Ownership Rules

When adding or changing anything, match it to the file that owns that concern. If you're unsure, use this table.

If you need to…Touch this fileDo not touch
Add or change a model name, URL, path, or context window size config.py Any other file — import from config instead
Change how the explain prompt is worded prompts.py explain.py — it only imports EXPLAIN_SYSTEM_PROMPT
Add an object to the inlet/outlet reference table prompts.py validate.py TRUSTED_PORT_COUNTS — keep both in sync if adding a new object
Change how patches are validated validate.py generate.py — it only calls validate_patch()
Change how ChromaDB is queried or embeddings are generated retrieve.py explain.py / generate.py — they only call query_vector_db()
Change the generate retry loop or example patch loading generate.py server.py — server only calls generate_patch()
Change how personal idioms are read or written guided.py server.py — server only calls the guided functions
Change HTTP routes, CORS, or WebSocket handling server.py Any assistant/ module — they must not know about HTTP
Change M4L context detection keywords explain.py server.py / generate.py — both call detect_m4l_context()

Request Flow

4.1 Explain (Q&A tab)

1 server.py receives POST /explain → calls explain_query() 2 explain.py calls query_vector_db() → retrieve.py returns chunks 3 explain.py calls load_inlet_outlet_index(), detect_m4l_context(), load_lom_schema() 4 explain.py builds user prompt, prepends EXPLAIN_SYSTEM_PROMPT from prompts.py 5 explain.py streams Ollama response → callback → server.py SSE to client

4.2 Generate (Patch Generator tab)

1 server.py receives POST /generate → calls generate_patch() 2 generate.py infers domain, calls query_vector_db() → retrieve.py 3 generate.py calls load_inlet_outlet_index(), load_lom_schema() (if M4L) from explain.py 4 generate.py builds prompt with GENERATE_SYSTEM_PROMPT from prompts.py └── GENERATE_SYSTEM_PROMPT contains the full 166-object inlet/outlet reference table 5 generate.py calls Ollama → extracts JSON → calls validate_patch() in validate.py 6 If invalid: appends errors to chat history, retries (max 3 attempts) 7 Returns {valid, patch, errors, attempts} → server.py → client

4.3 Guided Build (WebSocket)

1 server.py opens WebSocket /ws/guided 2 Each turn: server calls run_websocket_learning_stage() or run_websocket_generation_stage() └── These are async wrappers. Blocking Ollama calls use _ollama_post() via run_in_executor 3 guided.py provides GUIDED_SYSTEM_PROMPT, ENRICHED_USER_TEMPLATE, personal idioms 4 On disconnect: stop_event.set() cancels background streaming thread 5 End of session: append_to_personal_idioms() called to persist learnings

Data Files

All data files live under data/. Paths are defined in config.py and imported — never hardcoded in module files.

FileOwned byPurposeGit

Important: When adding new data file paths, define them in config.py first, then import in the module that reads the file. Never use relative path strings like "data/something.json" directly in module files — they break when the script is run from a different working directory.

Forbidden Patterns

These patterns caused the bugs found in the June 2026 code review. Do not reintroduce them.

PatternWhy it's forbiddenBug ref
Calling requests.post() inside async def without run_in_executor Blocks the entire asyncio event loop. No other requests can be handled while Ollama is thinking. #1
asyncio.create_task() without await in a fire-and-forget pattern Task may never complete on WebSocket disconnect. Silent data loss. #2
Spawning a background thread with no stop event Thread keeps running after WebSocket disconnects, wasting CPU and Ollama resources. #3
TEMPLATE.format(…, query=user_input) where user_input is raw Crashes with KeyError if user types { or } in their query. e.g. "how does {dict} work?" #4
b.maxclass.endswith("~") to detect DSP objects Misses newobj-wrapped audio objects. Use has_object() which checks both maxclass and text fields. #5
chromadb.PersistentClient() inside query functions Creates and tears down the DB client on every query. Use module-level singleton. #6
Opening inlet_outlet_index.json inside per-object validation calls File read repeated once per unknown object in the patch. Cache at module load. #7
allow_origins=["*"] combined with allow_credentials=True Invalid CORS configuration. Browsers reject it and OWASP flags it as a misconfiguration. #8
Defining constants that already exist in config.py Two copies drift apart silently. Import from config instead. #9
Hardcoded relative paths like "data/something.json" Breaks when run from any directory other than the repo root. #10
sys.path.insert() hacks in module files Fragile import system workaround. Use proper package structure with __init__.py instead. #11
Prompt strings defined inline in explain.py / generate.py / guided.py Scatters the prompt surface across files. All prompts live in prompts.py only. arch
